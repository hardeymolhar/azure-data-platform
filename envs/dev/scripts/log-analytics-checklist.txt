1. Always confirm the diagnostic category first

Run this before writing any query.

AzureDiagnostics
| summarize count() by Category

Why:
Ensures DataPlaneRequests exists before filtering on it.




2. Verify the columns you will use
AzureDiagnostics
| project Category, databaseName_s, collectionName_s, requestCharge_s
| take 10

Why:
]
Prevents "column not found" semantic errors.




3. Filter only Cosmos data-plane operations
| where Category contains "DataPlane"

Why:
These rows contain RU charges.




4. Remove empty system rows

Your logs contained entries like:

databaseName_s = ""
collectionName_s = ""

These are metadata operations, not container activity.

Always filter them:

| where isnotempty(databaseName_s)
| where isnotempty(collectionName_s)




5. Convert RU from string to numeric

requestCharge_s is stored as string.

Always convert before aggregation:

todouble(requestCharge_s)

Example:

| summarize totalRU=sum(todouble(requestCharge_s))




6. Use time binning when analyzing RU
bin(TimeGenerated,5m)

Why:

Without binning, large queries aggregate all RU and hide spikes.




7. When running from CLI, wrap the query correctly

Always use:

--analytics-query '
KQL QUERY
'

Rule:

Outer quotes → '
Inner quotes → "

This prevents errors like:

Failed to resolve scalar expression named 'DataPlane'
Final Stable Query (Your Working Version)
AzureDiagnostics
| where Category contains "DataPlane"
| where isnotempty(databaseName_s)
| where isnotempty(collectionName_s)
| summarize totalRU=sum(todouble(requestCharge_s))
  by bin(TimeGenerated,5m), databaseName_s, collectionName_s
| order by TimeGenerated desc
